# AI Game Master Rules

## Game Overview

AI Game Master is a cooperative game with AI-generated narrative, delivered through a Telegram bot. Each day generates a unique story in a space setting (starship, crew, adventures), where players make decisions that influence the plot development.

## Core Game Mechanics

### 1. Registration and Onboarding

- After session start (`/start`), player completes an interview to determine their role
- Interview consists of 2-3 questions like "what would you do in this situation" or "A or B choice"
- Questions are generated by game-master-api based on the general game setting
- Based on player answers, a personalized avatar is generated via comfyui
- Avatar and role description are sent to player after onboarding completion

### 2. Crew Assembly

- Multiple games can run simultaneously for different player groups
- A player can only participate in one game at a time
- Crew is assembled over 3 days from when the first crew member joined
- If team is not full by the end of the period, unfilled roles are assigned to NPC bots
- Each player receives a unique role (Chief Engineer, XO, Science Officer, etc.)

## Game Master (Ship Computer) Behavior

### At start of new game day:

1. **Game State Check**
   - Game Master API checks last game state
   - Verifies game hasn't ended (ship and crew alive)
   - If game is over, generation stops

2. **Story Generation**
   - Generates intro story for new day
   - Uses player actions from previous day
   - Adapts story based on choice consequences
   - Ensures story consistency and continuity

3. **Content Generation**
   - Via comfyui generates personalized comics
   - Each player receives their own story according to role
   - All stories unified by common day intro

4. **Player Action Handling**
   - Player selects action from offered options
   - Consequences recorded as input for next generation
   - If player doesn't select in time, AI Game Master chooses for them

## Player Communication

### Text Messages:

- Players can communicate with Game Master via text messages
- Messages used for text generation and responses
- Messages transferred to other players along with author's avatar
- Game Master responds in character as ship computer

### Voice Messages:

- Voice message support (no transcription yet)
- Recording and storage of voice messages
- Processing through Game Master API

## Game Mechanics

### Actions and Consequences:

- Each action has visible choice and hidden consequence
- Consequences influence story development in following days
- Consequence system creates sense of real impact on game world

### Team Play:

- Decisions made by group of players
- Affect overall story
- Creates sense of community and shared adventure

## Frequently Asked Questions

**Q: How often is new story generated?**
A: Every day at scheduled time (default 08:00)

**Q: What if I miss action selection?**
A: AI Game Master automatically selects for you based on your role and traits

**Q: Can I participate in multiple games simultaneously?**
A: No, player can only participate in one game at a time

**Q: How long does one game last?**
A: Game continues until ship and crew are alive. Ending depends on player decisions

**Q: Can I communicate with other players?**
A: Messages transferred through Game Master and visible to other players with avatars

## Conclusion

AI Game Master creates a unique daily experience where each player becomes part of a living story. Generated content, personalized comics, and cooperative mechanics create a sense of real space adventure.
