# Требования к AI Game Master (Нереализованные функции)

Этот документ содержит только функции, которые еще не реализованы в текущей системе.

## Критерии выполнения (Definition of Done)

### Game Master - Остальные требования

#### 1. Проверка состояния игры перед генерацией
- **Что реализовано:** Game master запускается по расписанию
- **Что нужно реализовать:**
  - Проверять последнее состояние игры перед генерацией нового дня
  - Проверять, что корабль и экипаж живы
  - Если игра закончена (корабль/экипаж погибли), прекратить генерацию и отправить финальное сообщение
  - Добавить в game_state таблицу поля для статуса корабля и здоровья экипажа

#### 2. Генерация персональных комиксов для каждого игрока
- **Что реализовано:** Инфраструктура для комиксов существует (comic_generator.py), но возвращает заглушку
- **Что нужно реализовать:**
  - Интегрировать Pixelle-MCP и ComfyUI для реальной генерации изображений
  - Генерировать персональные истории в виде комикса для каждого игрока в соответствии с его ролью
  - Объединить все истории общим вводным сюжетом дня
  - Каждая история должна содержать перечень действий (выборов) игрока
  - Сохранять сгенерированные комиксы и предоставлять URL доступа

#### 3. Генерация аватара игрока при онбординге
- **Что реализовано:** В базе данных есть поле avatar_description, но аватар не генерируется
- **Что нужно реализовать:**
  - Использовать npcpy и ComfyUI для генерации визуального аватара на основе выборов игрока при онбординге
  - Сохранять URL сгенерированного аватара в поле avatar_description таблицы player_profiles
  - Интегрировать аватар в профиль игрока в Telegram боте

#### 4. Использование последствий действий для следующей генерации
- **Что реализовано:** Инфраструктура для хранения действий существует, но последствия не используются в генерации следующего дня
- **Что нужно реализовать:**
  - При генерации нового дня учитывать последствия выбранных игроками действий из предыдущего дня
  - Формировать previous_summary на основе выбранных действий и их скрытых последствий
  - Обновить generate_daily_story() для использования истории выборов и их результатов
  - Создать систему влияния: как игроки влияют на мир через свои решения

## Заметки по реализации

### Интеграция с Pixelle-MCP
- Pixelle-MCP уже указан в docker-compose.yaml
- URL: http://pixelle-mcp:9004/pixelle/mcp
- Необходимо настроить MCP клиент для вызова инструментов генерации изображений

### Интеграция с NPCPY
- NPCPY библиотека для генерации персонажей
- Используется для создания уникальных личностей членов экипажа
- Необходимо интегрировать с ComfyUI для визуализации NPC

### Архитектурные соображения
- Все сервисы должны быть отдельными контейнерами в docker-compose
- Использовать асинхронные вызовы для генерации контента
- Хранить URL сгенерированных изображений в базе данных, не сами файлы
- Предоставить механизм повторной генерации при ошибках

## Следующие шаги

1. Реализовать проверку состояния игры
2. Настроить интеграцию с Pixelle-MCP для генерации изображений
3. Реализовать генерацию персональных комиксов
4. Добавить генерацию аватаров при онбординге
5. Интегрировать систему последствий действий в генерацию следующего дня
