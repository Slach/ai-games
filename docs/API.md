# Game Master API Documentation

## Base URL

```
http://game-server:8000
```

All game-specific endpoints accept `?game_id=...` query parameter (default: `"default_game"`).

## Image URLs

All images are generated by ComfyUI and served via its `/view` endpoint.
URL format: `http://comfyui:8188/view?filename=FILENAME&subfolder=SUBFOLDER&type=output`

`filename_prefix` passed to ComfyUI determines both the output filename and subfolder.
ComfyUI appends `_00001_` (batch counter) and `.png` automatically.

| Type | `filename_prefix` in code | Example resulting URL |
|------|--------------------------|----------------------|
| `avatar` | `{game_id}/avatar_{player_id}` | `http://comfyui:8188/view?filename=avatar_123456_00001_.png&subfolder=default_game&type=output` |
| `loading` | `{game_id}/loading_{i}` | `http://comfyui:8188/view?filename=loading_2_00001_.png&subfolder=default_game&type=output` |
| `splash` | `{game_id}/splash_{i}` | `http://comfyui:8188/view?filename=splash_2_00001_.png&subfolder=default_game&type=output` |
| `bridge` | `{game_id}/bridge` | `http://comfyui:8188/view?filename=bridge_00001_.png&subfolder=default_game&type=output` |
| `scene` | `{game_id}/scene_turn{turn}` | `http://comfyui:8188/view?filename=scene_turn3_00001_.png&subfolder=default_game&type=output` |
| `outcome` | `{game_id}/outcome_turn{turn}` | `http://comfyui:8188/view?filename=outcome_turn3_00001_.png&subfolder=default_game&type=output` |
| `finale` | `{game_id}/finale_{type}` | `http://comfyui:8188/view?filename=finale_victory_00001_.png&subfolder=default_game&type=output` |
| `character` | `{game_id}/char_turn{turn}_p{pid}` | `http://comfyui:8188/view?filename=char_turn3_p123456_00001_.png&subfolder=default_game&type=output` |
| `action` (player) | `{game_id}/action_turn{turn}_p{pid}` | `http://comfyui:8188/view?filename=action_turn3_p123456_00001_.png&subfolder=default_game&type=output` |
| `action` (NPC) | `{game_id}/action_turn{turn}_{npc_key}` | `http://comfyui:8188/view?filename=action_turn3_npc_captain_default_game_00001_.png&subfolder=default_game&type=output` |
| `onboarding_q` | `{game_id}/onboarding_q_{q_id}` | `http://comfyui:8188/view?filename=onboarding_q_2_00001_.png&subfolder=default_game&type=output` |
| `species_opt` | `{game_id}/species_{pid}_{q_id}_{opt_val}` | `http://comfyui:8188/view?filename=species_123456_7_s3_a_00001_.png&subfolder=default_game&type=output` |

How each type reaches the client:

- `avatar`, `loading`, `splash`, `bridge`, `scene` — stored in `game_images` table, retrieved via `get_random_game_image()`
- `action` — stored in `player_briefings.chosen_action_url` column, returned in `/game/poll/{player_id}`
- `character` — stored in push payload as `character_image_url`, not persisted in DB beyond `game_images`
- `outcome`, `finale` — stored in `game_images` table, pushed via `push_turn_outcome`/`push_game_over`
- `onboarding_q`, `species_opt` — embedded directly in question/option JSON via `image_url` field

### Onboarding

#### Start Onboarding Session

**POST** `/onboarding/start`

```json
{
  "player_id": 281412419,
  "game_id": "default_game",
  "language": "en",
  "player_name": "Alex"
}
```

**Response:**

```json
{
  "session_id": "abc123",
  "game_id": "default_game",
  "question": {
    "id": 1,
    "text": "Корабль обнаружил неизвестный сигнал. Ваши действия?",
    "image_url": "http://comfyui:8188/output/default_game/onboarding_q_1_00001_.png",
    "options": [
      {
        "value": "cautious",
        "label": "Изучить сигнал с осторожностью",
        "role_scores": {"captain": 1, "science_officer": 3, "chief_engineer": 0, ...}
      }
    ]
  },
  "game_title": "«Звёздный Странник: За гранью известного»",
  "welcome_message": "Добро пожаловать на борт «Звёздного Странника»..."
}
```

Questions are generated dynamically by LLM. Each option has `role_scores` dict (points per role key).
Optionally includes `image_url` if ComfyUI generates an illustration for the question, and
`image_prompt` (English prompt used for generation).

After the initial role-assignment questions, the onboarding continues with species/gender
questions (up to 5, alternating species/gender/species/gender/species). These are generated
one-at-a-time by the LLM as the player answers, with per-option images (`image_url` on each option)
showing cumulative visual effects of all previous choices.

### Game State

#### Get Game State

**GET** `/game/state?game_id=default_game`

```json
{
  "turn": 3,
  "status": "active",
  "ship_alive": true,
  "crew_health": 85,
  "last_updated": "2026-06-27T12:00:00",
  "last_death_turn": 0,
  "finale_narrative": "",
  "finale_outcome_type": "",
  "finale_image_url": ""
}
```

- `status`: `"active"` while the game is running; changes to `"ship_destroyed"`, `"mission_complete"`, `"crew_wiped"`, or `"game_over"` when the game ends
- `turn`: the **next** turn to generate (current completed turn = `turn - 1`)
- `finale_*` fields are empty during active play; populated when the game ends with LLM-generated finale

#### Get Game Finale

**GET** `/game/finale?game_id=default_game`

Returns the LLM-generated game-over finale.  Only available after the game ends.
Returns **404** if the game is still active or no finale was generated.

```json
{
  "game_id": "default_game",
  "status": "ship_destroyed",
  "finale_narrative": "Корабль «Атлас-9» погиб в неравном бою...",
  "finale_outcome_type": "defeat",
  "finale_image_url": "http://comfyui:8188/view?filename=finale_defeat_00001_.png&subfolder=default_game&type=output"
}
```

- `finale_outcome_type`: `"victory"` (mission completed) or `"defeat"` (ship destroyed / crew wiped)
- `finale_image_url`: finale illustration generated by ComfyUI (may be empty)

### Game Turns (personal briefing system)

The game flow is **per-player personal briefings**, not a shared story.
Each player gets their own briefing text and action choices.

#### Get Current Game Turn

**GET** `/game/current-turn?game_id=default_game`

Returns the latest completed turn's database record (legacy format).

```json
{
  "turn": 2,
  "story": "Экипаж приближается к аномалии...",
  "global_circumstances": "{\"setting\":\"...\",\"conflict\":\"...\",\"narrative\":\"...\",\"key_events\":[...]}",
  "crew_dialogues": [{"npc": "Captain Eva Rodriguez", "dialogue": "Всем занять свои посты!"}],
  "player_actions": [],
  "generated_content": {"image": "/content/turn_2/scene.jpg"},
  "previous_turn_summary": "...",
  "combined_outcome": "{\"outcome_narrative\":\"...\",\"ship_hull_integrity\":95,...}"
}
```

> **Note:** `generated_content.image` is a **legacy placeholder** and is never served to
> clients. Actual images come from the `/game/poll/{player_id}` endpoint and content
> endpoints (`/content/loading-image`, `/content/splash-image`, `/game/bridge-image`).

#### Get Specific Game Turn

**GET** `/game/turn/{turn_num}?game_id=default_game`

Same shape as `/game/current-turn`.

#### Poll Game Updates (primary client endpoint)

**GET** `/game/poll/{player_id}?since=2026-06-27T12:00:00`

This is the **main endpoint** that Telegram clients poll to get new turns, briefings, and messages.

```json
{
  "new_game_turn": null,
  "pending_actions": [],
  "personal_briefing": {
    "briefing": "Ты, Алекс, главный инженер, стоишь у панели управления...",
    "choices": [
      {
        "id": "action_1",
        "text": "Починить варп-двигатель вручную",
        "consequence": "Двигатель восстановлен, но корабль теряет щиты на 10%"
      },
      {
        "id": "action_2",
        "text": "Запросить помощь у научного отдела",
        "consequence": "Решение найдено совместно, но требует времени"
      }
    ],
    "chosen_action_url": null,
    "briefing_image_url": "http://comfyui:8188/output/default_game/scene_turn2_00001_.png",
    "crew_dialogues": [
      {"npc": "Captain Eva Rodriguez", "dialogue": "Инженер, доложите статус!"}
    ]
  },
  "messages_from_gm": [
    {"id": 1, "player_id": 281412419, "message": "...", "message_type": "text", "timestamp": "..."}
  ],
  "npc_messages": []
}
```

- `personal_briefing` is present when the player has a pending choice for the current turn.
- `chosen_action_url` is set after the player submits an action and the image is generated.
- `briefing_image_url` is the scene image for this turn (generated by ComfyUI).
- `pending_actions` mirrors `choices` for backward compatibility.

### Player Actions

#### Submit Player Action

**POST** `/game/actions`

```json
{
  "player_id": 281412419,
  "turn": 2,
  "action_id": "action_1",
  "choice": "selected"
}
```

**Response:**

```json
{"status": "accepted", "action": {...}}
```

After submission, a comic-style action image is generated in the background
(ComfyUI with player's avatar as reference) and pushed to the player when ready.
When **all** players have chosen, combined outcome analysis runs automatically.

#### Auto-Select Action

**POST** `/game/auto-action/{player_id}/{turn}?game_id=default_game`

Auto-selects an action for a player who hasn't chosen in time.
Uses LLM with global circumstances, personal briefing, and player profile.
Notifies the player about the auto-selection via game messages.

```json
{
  "status": "selected",
  "action_id": "action_1",
  "action_text": "Починить варп-двигатель вручную",
  "rationale": "As Chief Engineer, maintaining ship systems is the top priority..."
}
```

Returns `{"status": "already_chosen", "action_id": "..."}` if the player already chose.

#### Get Player's Personal Briefing


#### Get Bridge Image (or Scene Image)

**GET** `/game/bridge-image?game_id=default_game&turn=2`

With `turn=N`: returns the scene image for that specific turn.
Used by `cmd_turn` in telegram-bot to show a scene illustration alongside the story.

```json

  "image_url": "http://comfyui:8188/output/default_game/bridge_00001_.png",

}
```

#### Start Game

**POST** `/admin/start-game`

```json
{"game_id": "abc123", "language": "ru", "force": true, "was_restarted": false}
```

Starts background generation (NPCs, mission, bridge image, first turn briefings).
Returns immediately with `{"status": "accepted", ...}`. GM receives Telegram notification when complete.

What happens:

1. Creates NPCs for all unfilled crew roles (with LLM-generated names, randomized species/gender)
2. Generates NPC avatars via ComfyUI
3. Generates mission via LLM
4. Generates bridge image via ComfyUI
5. Generates global circumstances for turn 1 via LLM
6. Generates per-player personal briefings and choices (players + NPCs in parallel)
7. NPCs auto-select their actions immediately
8. Generates per-player character-in-scene images
9. Generates NPC dialogues
10. Pushes briefings to all players via telegram-bot

#### Continue Game (next turn)

**POST** `/admin/continue-game?game_id=abc123&language=ru&force_resend=false`

Generates the next turn. Runs in background, returns immediately.
GM receives Telegram notification when complete.

Similar flow to start-game, plus:

- Builds cumulative story summary from all previous turns
- Before pushing new briefings, analyzes and delivers the previous turn's outcome
  (combined outcome of all player/NPC decisions, ship damage, injuries, deaths)

**POST** `/admin/analyze-turn?turn=2&language=ru&game_id=abc123`

#### Regenerate Turn

**POST** `/admin/regenerate-turn?game_id=abc123&language=ru`

Deletes the current turn's data (briefings, actions, turn record),
rolls back game state by one turn, then regenerates.
Runs in background, returns immediately.

#### Restart Game

**POST** `/admin/restart-game?game_id=abc123&language=ru`

Full wipe of game content while keeping player profiles:

- Deletes all game turns, briefings, actions, messages, mission, images
- Resets game state to turn 1
- Marks game as not started
- Makes all roles available again
- Deactivates all NPCs (fresh ones will be generated on next start)

#### Kick Player

**POST** `/admin/kick-player`

```json
{"role_key": "chief_engineer", "reason": "GM is god ;)", "game_id": "abc123", "language": "ru"}
```

Replaces the player with an NPC (preserving the player's name, species, gender, and appearance),
releases the role, notifies the kicked player, and updates the database.

#### Reset Player

**POST** `/admin/reset-player`

```json
{"player_id": 281412419, "language": "ru"}
```

Replaces the player with an NPC (if they hold a role), then deletes their profile
and onboarding sessions, allowing them to start over.

#### Generate Loading Images

**POST** `/admin/generate-loading-images?count=5&game_id=abc123`

```json
{"status": "success", "requested": 5, "generated": 5, "saved": 5, "total_in_db": 5}
```

#### Generate Splash Images

**POST** `/admin/generate-splash-images?game_id=abc123&lang=ru`

Generates 3 splash images using the current game title.

## Game Scheduler

**Base URL:** `http://game-scheduler:8001`

The scheduler manages per-game turn generation intervals. Games are **registered automatically**
by the game-server when >= 3 players join (game starts). The GM can override schedules via
`/gm_schedule` in the Telegram bot.

### Registration & Lifecycle

| Event | What happens |
|-------|-------------|
| Game starts (>= 3 players join) | Game-server calls `POST /scheduler/register/{game_id}` with default schedule |
| Game ends (ship destroyed / mission complete / crew wiped) | Scheduler sets `mode: "ended"` — schedule is **preserved** in DB for restart |
| GM runs `/gm_schedule <game_id> 10m` | Telegram bot calls `POST /scheduler/schedule/{game_id}` with new schedule |
| Scheduler restart | Loads all persisted schedules from DB; ended/paused games are skipped |

### Endpoints

#### Scheduler Status

**GET** `/scheduler/status?game_id=default_game`

Without `game_id`: returns status for all registered games.
With `game_id`: returns single game's status.

```json
{
  "game_id": "default_game",
  "mode": "scheduled",
  "schedule_type": "interval",
  "schedule_value": "600",
  "last_run_at": "2026-06-28T06:30:00+00:00",
  "next_run_at": "2026-06-28T06:40:00+00:00"
}
```

Modes: `"scheduled"` (active), `"paused"` (GM paused), `"ended"` (game finished).

#### Register Game

**POST** `/scheduler/register/{game_id}?schedule=10m`

Registers a game with an optional schedule override. Called automatically by
game-server when the game starts.

#### Set Schedule

**POST** `/scheduler/schedule/{game_id}`

```json
{"schedule": "6h"}
```

Schedule formats: `"Nh"`, `"Nm"`, `"Ns"` (interval) or `"HH:MM"` (daily at that time).

#### Pause / Resume

**POST** `/scheduler/pause?game_id=default_game`
**POST** `/scheduler/resume?game_id=default_game`

#### Force Generate Now

**POST** `/scheduler/generate-now?game_id=default_game`

Immediately triggers a turn generation for the given game, bypassing schedule.
